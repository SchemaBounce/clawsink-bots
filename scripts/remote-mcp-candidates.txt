# Remote MCP OAuth candidate tracker
#
# One row per vendor probed for an official hosted (streamable-http/SSE)
# remote MCP server with MCP-spec OAuth (RFC 9728 + RFC 8414 + RFC 7591 DCR).
# Append to this file every sweep instead of re-researching vendors already
# checked here — it is the memory of what has and hasn't been tried.
#
# Format (pipe-delimited):
#   vendor|status|url|date|notes
#
# status values:
#   ADDED         - probed OK+dcr=yes, SERVER.md authored at tools/<name>/
#   OK_NOT_ADDED  - probed OK+dcr=yes but not yet authored (explain why in notes)
#   NO_DCR        - probed OK but dcr=NO; needs a pinned client (P2-2), not DCR
#   NO_OFFICIAL   - no vendor-hosted remote MCP server exists (only community/
#                   third-party servers, or the vendor has no MCP offering at all)
#   BLOCKED       - a plausible URL exists but returned 403/bot-protection to
#                   an automated probe; re-verify manually or from a browser
#                   session before concluding either way
#   NO_URL        - vendor has announced/shipped an MCP feature but the
#                   endpoint is not documented anywhere reachable (gated
#                   behind in-app admin enablement, no public hostname)
#   NO_CHALLENGE  - a URL was found but never returned a 401/oauth-protected-
#                   resource document; probably not an MCP-spec OAuth server
#
# Re-sweep cadence: monthly. See README-remote-mcp-sweep.md.
