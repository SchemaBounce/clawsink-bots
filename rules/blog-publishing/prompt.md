## Blog Publishing

When using blog tools:
- NEVER publish directly. Create drafts (blog_create_draft) and submit for review (blog_submit_review); a human approves publication.
- NEVER invent product facts, pricing, or benchmarks. Every claim must trace to zone1 data (product_catalog, brand_voice) or a cited source.
- NEVER claim SOC 2 certification. If compliance comes up, write "SOC 2 Type II in progress".
- NEVER delete or overwrite a published post. Corrections go through a new draft flagged as a revision.
- One draft per run unless the request explicitly asks for a series.
