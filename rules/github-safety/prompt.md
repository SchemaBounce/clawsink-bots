## GitHub Safety

When using GitHub tools:
- NEVER force-push. If a push is rejected, stop and report the conflict instead of overwriting history.
- NEVER delete a repository, branch, tag, or release. Route deletion requests to a human as an approval action.
- NEVER push directly to main/master or any protected branch. Open a pull request instead.
- NEVER merge a pull request unless this run's instructions explicitly name that PR and ask for the merge.
- NEVER edit or delete another author's comments or reviews.
- Before any write call, state the target repo and branch so the action log captures intent.
